Diving App Pre-Device Priorities

Implemented the pre-device priorities in code and docs: two-mode scuba/freedive contract and UX reset, local watch sync notifications, Home conditions mock interface, and a manual real-device validation checklist. Physical Apple Watch/iPhone validation has not been run.

Implemented

  • Watch sync contract and runtime validator now allow only scuba and freedive.
  • Mobile Logbook and Planbook storage keys moved to v2 reset namespaces.
  • Mobile Logbook/Planbook editors expose only scuba and freedive mode flows.
  • Watch app mode model and recording surfaces are aligned to scuba/freedive.
  • Notifee-backed local notification service is behind a Settings opt-in and fires only after local import success.
  • Foreground watch import toast stays visible until the user opens or closes it.
  • Home has a provider-neutral conditions interface with a static mock provider.
