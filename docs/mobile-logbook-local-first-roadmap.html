Diving App Roadmap · 2026-06-21

모바일 로그북을 local-first로 세우고 Supabase sync로 확장한다

모바일 앱은 로그북의 중심이 되고, 워치 앱은 측정 가능한 세션 데이터를 제공하는 캡처 소스가 된다. 첫 구현은 로그인 없는 로컬 저장으로 시작하되, 인증과 Supabase 동기화가 나중에 같은 모델 위에 붙도록 설계한다.

Final Goal

사용자는 워치 없이도 모바일에서 로그를 만들 수 있고, 워치 세션을 가져온 경우에는 측정값을 보존한 채 부족한 맥락을 모바일에서 채워 최종 로그를 작성한다.

Storage Goal

비로그인 사용자는 기기에만 저장한다. 로그인 사용자는 같은 로컬 저장 흐름을 유지하면서 Supabase에 백업과 동기화를 수행한다.

Safety Goal

앱은 과거 기록, 리뷰, 비중요 계획 알림만 다룬다. 감압, 조직 포화도, 비상 판단, 인증된 다이브 컴퓨터 주장은 범위 밖이다.

Architecture

화면은 저장 방식에 의존하지 않고, 저장소 인터페이스가 로컬과 Supabase 구현을 바꿔 끼우는 구조를 만든다.

Mobile Screens Logbook list, detail, manual editor, watch review editor. UI는 provenance와 편집 가능 여부만 표시한다.
State Hooks useDiveLogbook, future useDiveLogEditor. 화면 이벤트를 도메인 action으로 바꾼다.
Repository Interface DiveLogRepository. list/get/save/delete/import watch session을 제공한다.
Local Repository 첫 구현. guest mode와 offline cache의 기반이다.
Syncing Repository 후속 구현. 로그인 상태에서 local write 후 Supabase sync를 수행한다.

Data Model Direction

type DiveLogSource = 'manual' | 'watch';
type DiveFieldSource = 'manual' | 'mobile' | 'watch';
type DiveLogSyncStatus = 'localOnly' | 'pending' | 'synced' | 'failed';

type DiveLogEntry = {
  localId: string;
  remoteId?: string;
  ownerUserId?: string;
  source: DiveLogSource;
  syncStatus: DiveLogSyncStatus;
  manual: DiveLogManualFields;
  mobileAssisted?: DiveLogMobileAssistedFields;
  watch?: DiveLogWatchCapture;
};

Field Provenance

manual · editable mobile · suggested watch · locked source

워치 측정값은 원본으로 보존하고 수정하지 않는다. 사용자가 문제를 느끼면 원본을 덮어쓰기보다 메모, 표시 제외, 후속 보정 계층으로 다룬다.

Phased Plan

전체 로드맵은 6단계다. 세부 task는 지금 바로 실행 가능한 Phase 0-2에 가장 촘촘히 둔다.

Phase 0

Investigation

현재 mobile logbook, watch JSON export, local persistence 후보를 확인하고 첫 구현 file list를 확정한다.

  • 로컬 저장 엔진 후보 비교
  • 수동 로그 첫 field set 확정
  • fixture import를 repository action으로 옮길 범위 확정
Phase 1

Domain and Repository Contract

WatchSession과 분리된 DiveLogEntry를 만들고 저장소 인터페이스를 도입한다.

  • source, syncStatus, field provenance 정의
  • manual draft 생성 helper 작성
  • watch session to log entry mapper 작성
Phase 2

Local Manual Log Creation

로그인 없이 모바일에서 로그를 만들고 저장한다. 이 기능은 나중에도 guest mode와 offline cache로 유지된다.

  • Logbook create action
  • manual editor
  • local repository save/list/detail
Phase 3

Watch Import Validation

워치에서 contract-valid payload가 모바일까지 도달할 수 있는지 실제 transport 또는 debug bridge로 검증한다.

Phase 4

Watch-Based Log Authoring

워치 측정값은 잠금 처리하고, 모바일에서 site, buddy, marine life, notes 같은 누락 정보를 채워 최종 로그를 만든다.

Phase 5

Auth and Supabase Foundation

Supabase Auth, user-owned tables, RLS, generated types, remote repository를 추가한다.

Phase 6

Login-Aware Storage Strategy

비로그인 사용자는 local-only, 로그인 사용자는 local write 후 Supabase sync로 저장한다.

Next Detailed Tasks

1. Local persistence decision key-value JSON과 SQLite 계열 후보를 비교하고 첫 구현에 맞는 저장소를 고른다.
2. DiveLogEntry contract manual, mobile-assisted, watch-captured field를 분리하는 TypeScript type을 만든다.
3. Repository interface 화면이 저장 엔진을 모르게 만드는 DiveLogRepository를 추가한다.
4. Watch mapper WatchSession을 locked watch-sourced DiveLogEntry로 변환한다.
5. Manual editor site, time, duration, max depth, buddy, tags, marine life, notes, rating을 먼저 지원한다.
6. List/detail integration manual log와 watch log가 같은 로그북에 보이되 provenance와 편집 가능 여부를 다르게 보여준다.

Safety Boundary

이 계획은 다이빙 기록과 사후 리뷰를 위한 것이다. 워치에서 가져온 값은 워치 측정값 또는 watch-captured 출처로 표시할 수 있지만, 인증된 장비의 안전 판단이나 감압 계산처럼 표현하지 않는다.
